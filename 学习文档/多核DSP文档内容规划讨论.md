## 我理解的架构

**角色分工**

CORE0 是"大脑 + 感官 + 嘴巴"：它负责感知外界（读传感器）、决策调度（踢从核）、对外说话（发 FPGA）。它是**唯一和外设打交道**的核——EDMA3 采集、SRIO 发送、以太网通信、CAN/RS485 都归它。从核完全被隔离在外设之外，这是一种很干净的**职责分离**。

CORE1/2/3/7 是"四条并行的计算流水线"：它们**不和任何外设交互**，只做纯计算。输入是共享内存里的一组数据（来自 CORE0 预处理过的传感器值），输出是共享内存里的另一组数据（控制量）。从核存在的唯一理由就是**并行算力**——把原本 CORE0 一个核要串行算完的多轴控制律，分摊到四个核上同时算，把单周期计算时间压缩到 1/4。

**时序上的"心跳"模式**

整个系统由一个 Timer 中断驱动，这就是"心跳"。每一次心跳触发一个完整的控制周期：

1. **Timer 中断触发** → CORE0 的 `IsrMotionControl()` 开始执行
2. **感知阶段**：CORE0 通过 EDMA3 采集传感器 → 等 EDMA 完成 → AD 预处理 → 光栅尺解算 → 结果写进 MSMC 共享内存
3. **派发阶段**：CORE0 把 `Phase[1..7]` 标记成"待处理"状态，然后 `KickUnlock` → 同时触发 4 个 IPC 中断唤醒 CORE1/2/3/7 → `KickLock`
4. **并行计算阶段**：四个从核**同时**从共享内存读自己负责的那份数据，各算各的控制律，算完把结果写回共享内存，把自己的 `Phase[x]` 标回 `Free`
5. **汇总阶段**：CORE0 在后续代码里轮询 `CheckAllAxisFinish()`，等所有从核都 Free 之后，从共享内存收齐所有控制量
6. **输出阶段**：`VxDooBellSend()` 通过 SRIO DoorBell 把汇总好的控制量发给 FPGA，FPGA 执行
7. 等下一个 Timer 中断 → 回到第 1 步

**这个模式的精妙之处**

- **从核代码极其简单**：一个 IPC 中断服务程序 + 一段算法代码 + 一次 `Phase[x] = Free`，几乎没有状态机、没有通信栈、没有驱动，所以 CORE1/7 的目录才那么精简（没有 ethernet、没有 can、没有 fpga）——不是"省下来了"，是**根本不需要**。
- **所有外设访问都在 CORE0**，所以不存在"两个核同时抢一个外设"的问题，不需要复杂的外设锁。
- **时序高度确定**：每个控制周期的事件顺序固定，CORE0 的关中断/开中断、IPC 触发、SRIO 发送都发生在确定的时间点，抖动小。
- **共享内存只在明确的时间点被读写**：CORE0 写的时候从核一定没在读（因为从核还没被踢醒），从核写的时候 CORE0 在等待（在 `CheckAllAxisFinish` 自旋），所以**天然避免了读写冲突**，不需要互斥锁。这就是为什么你能用 `Phase[]` 这种看起来很"土"的标志位做同步——因为时序本身保证了安全。

**`KickUnlock/KickLock` 的小细节**

这对函数包住 IPC 触发，是 C6678 的 "Kicker" 机制——某些关键寄存器（IPC 生成中断寄存器也在其列）默认是锁住的，防止误写。要写之前必须先往 Kicker 寄存器填两个魔数（`0x83E70B13` 和 `0x95A4F1E0`，你的 `hw_C6678.h` 里就有 `KICK0_UNLOCK`/`KICK1_UNLOCK`）解锁，写完再锁上。这是 TI 的安全设计，文档里会单独讲一下，挺有意思的一个小知识点。

**Cache 一致性在这里为什么重要**

因为共享内存在 MSMC（`0x0C000000`），而每个核都有自己的 L1D 和 L2 Cache。如果 CORE0 把传感器数据写进共享内存，但这数据还卡在 CORE0 的 L2 Cache 里没刷回 MSMC，那 CORE1 读到的就是**旧数据**。反过来，CORE1 算完写到共享内存，但 CORE0 如果之前读过同一地址，CORE0 的 Cache 里会有一份**陈旧的副本**，CORE0 再读时会命中自己的 Cache，拿到的是旧值。

所以每个核才都有 `cache/drv_cache.c`——CORE0 在踢从核**之前**要对写入区 `writeback`，从核被唤醒时要对读入区 `invalidate`；从核写完要 `writeback`，CORE0 在 `CheckAllAxisFinish` 读 `Phase[]` 之前也要 `invalidate`。这些 Cache 操作会在第 7 章专门展开。
